《文本挖掘》课程期末大作业要求

本学期《文本挖掘》课程即将结束，为了全面考察大家对课程知识的掌握程度，特别是将理论应用于实践的能力，现将期末大作业的具体要求布置如下。请大家仔细阅读，并按时完成。

一、 作业核心任务

近年来，大语言模型（Large Language Model, LLM）为自然语言处理（NLP）领域带来了范式性的变革。其核心特征在于，能够以一个统一的模型和“提示” (Prompting) 的范式来处理极其多样化的NLP任务，而不再需要像传统方法那样为每个任务单独设计和训练专门的模型。

那么，在这种新范式下，大语言模型在各类具体的NLP任务上表现究竟如何？相较于为特定任务精细优化的传统方法，它存在哪些优势与不足？

本次期末作业的核心，就是要求大家通过实证研究来回答这一前沿问题。每位同学需要：

  1. 独立选择一个中文语言理解评测基准（CLUE）中的具体任务。
  2. 调用大语言模型API，以“提示”的方式来解决这个任务。
  3. 将其结果与该任务上经典的非大语言模型方法的公开成果进行严格的量化比较和深入分析，从而亲身体验并评估这一技术变革的实际效果与局限。

二、 具体要求与步骤

1. 任务选择

  • 访问CLUE任务列表: https://www.cluebenchmarks.com/introduce.html
  • 从列表中选择一个你感兴趣的任务进行研究。

2. 文献研究与理论阐述

在动手实验之前，你需要进行充分的文献调研，并清晰地阐述以下内容：

  • 任务定义: 准确描述你所选任务的目标是什么。
  • 任务难点: 分析该任务在技术和理论层面存在的主要挑战。
  • 现有方法: 总结学术界已经提出的、用于解决该任务的主流方法或模型。
  • 商业应用: 结合所学商科知识，探讨该任务在大数据管理与应用、数字经济、国际商务、会计学等领域的实际应用场景和商业价值。

3. 实验操作：调用大语言模型API

  • 数据下载: 在你所选任务的介绍页面，下载官方提供的测试集（Test Set）。
  • API调用: 选择一个大语言模型（如DeepSeek, Qwen, 豆包, Kimi等），通过调用其API，对测试集中的每一条数据进行预测，并保存完整的预测结果。
    • 模型选择: 可参考大语言模型综合能力评测平台：https://www.superclueai.com/
    • API资源:
      • 课程提供免费的DeepSeek API Key，使用方法请参考Kaggle上的示例代码：DeepSeek API Demo。
      • 若选择其他模型，可能需要自行注册并获取API Key。

4. 结果评估与对比分析

  • 确定评价指标: 查找提出该数据集的原始论文或使用该数据集的相关研究，明确该任务使用的评价指标（如准确率Accuracy, 精确率Precision, 召回率Recall, F1值等）。
  • 计算性能: 编程计算你选择的大语言模型在测试集上的各项评价指标得分。
  • 对比分析:
    • 找到一篇（或多篇）使用了相同测试集的非大语言模型方法的研究论文。
    • 将你的大语言模型结果与论文中报告的结果进行对比，制作清晰的图表来展示性能差异。
    • 务必确保你的实验设置（尤其是测试集）与所参考的论文完全一致。

5. 错误分析 (Error Analysis)

对大语言模型预测失败的案例进行深入分析。你需要：

  • 挑选一些典型的错误样本。
  • 分析模型出错的可能原因（例如，语义理解偏差、知识缺失、逻辑推理错误等）。
  • 思考改进的方向。

    1. 任务定义

    4. 商业应用场景探讨

    7. 核心API调用代码及返回结果示例
    8. 实验结果对比与分析（包含图或表）
    9. 大语言模型错误分析
